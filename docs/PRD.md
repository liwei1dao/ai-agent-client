**项目名称**：AI Agent Client
**版本**：v1.1
**日期**：2026-03-12
**状态**：草稿

---

## 1. 项目背景与目标

### 1.1 背景

随着大语言模型（LLM）、语音识别（STT）、语音合成（TTS）等 AI 能力的快速发展，各大厂商（微软、Google、阿里、火山引擎/豆包、OpenAI 等）均提供了成熟的 API 服务。然而在实际开发中，不同厂商接口差异大、集成成本高、切换成本高，缺乏一个统一的测试和验证平台。

### 1.2 目标

构建一个 **Flutter 全前端 AI 能力测试平台**，核心目标：

- 以插件化、接口化方式统一管理多厂商 AI 服务（STT/TTS/LLM/Translation）
- 通过 Agent 服务对象（chat-agent、translate-agent 等）编排 AI 能力
- Flutter 仅作为展示层和输入层，业务逻辑全部在 Agent 层处理
- 快速验证不同 AI 服务组合的效果，支持热切换 Provider
- 所有配置通过环境变量文件管理，无需后端服务

### 1.3 核心价值

| 价值点 | 描述 |
|--------|------|
| 快速验证 | 切换不同 AI Provider 只需修改配置，无需改代码 |
| 统一规范 | 所有 Provider 遵循统一接口，业务逻辑与服务商解耦 |
| 可扩展 | 新增 Provider 或 Agent 按规范接入，不影响现有功能 |
| 全前端 | 无需部署后端服务，配置写入 .env 文件即可运行 |

## 2. 用户画像

**主要用户**：AI 应用开发者、技术评估人员

**使用场景**：
- 评估不同厂商 STT/TTS 的识别/合成质量
- 测试不同 LLM 的对话能力与延迟
- 验证语音对话（STT + LLM + TTS）的端到端流程
- 测试翻译服务质量与速度
- 作为新 AI 项目的基础脚手架快速启动

## 3. 功能模块

### 3.1 模块总览（3-tab 导航）

```
AI Agent Client（底部导航）
├── 🤖 Agents（Agent 面板）[默认]
│   ├── Agent 列表（卡片展示，按类型色标区分）
│   ├── 添加/编辑 Chat Agent（选服务 + 系统提示词 + TTS 音色）
│   ├── 添加/编辑 Translate Agent（选服务 + 语言对 + TTS 音色）
│   ├── Chat Agent 运行界面（文本/语音对话）
│   └── Translate Agent 运行界面（文本/语音翻译）
├── 🧪 Services（服务库）
│   ├── 已配置服务列表（STT/TTS/LLM/翻译）
│   ├── 添加/编辑/删除服务
│   └── 服务快速测试（STT/TTS 连接验证）
└── ⚙️ Settings（全局设置）
    ├── 后台服务控制
    ├── 对话默认参数（历史长度/温度/Max Tokens）
    └── 语音默认参数（采样率/语速/音调）
```

### 3.2 Agent 面板（Agents Tab）

**核心概念**：用户创建多个 Agent，每个 Agent 是一个命名的 AI 能力配置。对话和翻译都是 Agent 类型，未来可扩展更多类型。

**Agent 管理**：

| 功能 | 优先级 | 描述 |
|------|--------|------|
| Agent 列表 | P0 | 网格/列表展示所有已创建的 Agent，色标区分类型 |
| 创建 Chat Agent | P0 | 选择 LLM（必选）、STT（可选）、TTS（可选），填系统提示词，选 TTS 音色 |
| 创建 Translate Agent | P0 | 选择翻译服务（必选）、语言对，配置 TTS 朗读（可选）及音色 |
| 编辑/删除 Agent | P1 | 长按或菜单编辑、删除已有 Agent |
| Agent 类型扩展 | P2 | 未来支持更多 Agent 类型（如摘要、代码审查等）|

**输入模式（聊天和翻译均支持三种）**：

| 模式 | 优先级 | 描述 |
|------|--------|------|
| 文本模式 | P0 | 键盘输入，手动点击发送 |
| 短语音模式 | P0 | 按住录音按钮说话，松开后 STT 识别并自动发送（类微信语音）；上滑可取消 |
| 通话模式 | P1 | 持续监听，VAD 检测静音后自动触发 STT→处理→TTS，播完后自动进入下一轮监听；支持随时打断 |

**Chat Agent 运行**：

| 功能 | 优先级 | 描述 |
|------|--------|------|
| 文本对话 | P0 | 文本模式：输入文本，LLM 流式回复 |
| 短语音对话 | P0 | 短语音模式：按住录音 → STT → LLM → 可选 TTS |
| 通话对话 | P1 | 通话模式：VAD 自动触发，连续免持对话，支持打断 |
| 流式输出 | P0 | LLM 回复流式（SSE）显示，含打字机光标 |
| 对话历史 | P0 | 完整对话历史，Markdown 渲染支持 |
| 实时 STT 预览 | P1 | 短语音/通话模式下显示实时识别文字 + 波形动画 |
| 模式切换 | P0 | 界面顶部 Tab 随时切换输入模式，通话模式有专用状态栏 |

**Translate Agent 运行**：

| 功能 | 优先级 | 描述 |
|------|--------|------|
| 文本翻译 | P0 | 文本模式：输入原文，返回译文 |
| 短语音翻译 | P0 | 短语音模式：按住录音 → STT → 翻译 → 可选 TTS 朗读 |
| 同传模式 | P1 | 通话模式：持续监听，自动逐句翻译并朗读，适合实时口译场景 |
| 语言对切换 | P0 | 源语言/目标语言选择，支持一键互换 |
| 语言自动检测 | P1 | 源语言设为 auto 时自动检测 |
| 译文朗读 | P1 | TTS 合成目标语言语音播放 |

### 3.3 服务库（Services Tab）

**核心概念**：服务配置一次全局可用，多个 Agent 可复用同一服务配置，无需重复填写 API Key。

| 功能 | 优先级 | 描述 |
|------|--------|------|
| 服务列表 | P0 | 展示所有已配置服务（类型/厂商/连接状态）|
| 添加服务 | P0 | 选类型→选厂商→填 API Key→命名保存 |
| 测试连接 | P0 | 验证 API Key 是否有效，展示连接状态 |
| 编辑/删除服务 | P1 | 修改 API Key 或删除服务 |
| STT 快速测试 | P1 | 快速录音验证 STT 服务可用性 |
| TTS 快速测试 | P1 | 快速合成验证 TTS 服务可用性 |

### 3.4 Agent 配置详情

**Chat Agent 配置项**：

| 配置项 | 必选 | 说明 |
|--------|------|------|
| Agent 名称 | ✅ | 用户自定义显示名称 |
| LLM 服务 | ✅ | 从服务库选择 LLM 服务 |
| STT 服务 | ❌ | 从服务库选择（启用语音输入）|
| TTS 服务 | ❌ | 从服务库选择（启用语音输出）|
| TTS 音色 | ❌ | TTS 服务选中后展开，选择具体音色 Chip |
| 系统提示词 | ❌ | 自定义 system prompt，支持多行 |
| 历史消息数 | ❌ | 默认 20，可覆盖全局设置 |

**Translate Agent 配置项**：

| 配置项 | 必选 | 说明 |
|--------|------|------|
| Agent 名称 | ✅ | 用户自定义显示名称 |
| 翻译服务 | ✅ | 从服务库选择翻译服务 |
| 源语言 | ✅ | 默认 auto（自动检测）|
| 目标语言 | ✅ | 默认中文 |
| TTS 服务 | ❌ | 从服务库选择（用于译文朗读）|
| TTS 音色 | ❌ | TTS 服务选中后展开，按目标语言筛选 |

### 3.5 全局设置（Settings Tab）

| 配置项 | 描述 |
|--------|------|
| 后台运行 | Android ForegroundService / iOS Background Audio 开关 |
| 电池优化 | 引导用户忽略电池优化（Android）|
| 历史消息数（默认）| 全局默认值，Agent 可覆盖 |
| Temperature（默认）| LLM 随机性参数 0.0~2.0 |
| Max Tokens（默认）| LLM 最大输出长度 |
| Markdown 渲染 | 聊天界面是否渲染 Markdown |
| STT 采样率 | 16000 / 8000 Hz |
| TTS 语速 | 0.5x~2.0x |
| TTS 音调 | -10~10 |

## 6. 非功能需求

### 6.1 性能要求
- STT 识别延迟 < 2s（网络正常情况下）
- LLM 首字延迟 < 3s（流式模式）
- TTS 合成延迟 < 1.5s
- 翻译响应延迟 < 2s

### 6.2 扩展性要求
- 新增 Provider 只需实现对应接口，注册到工厂即可
- 新增 Agent 只需继承 BaseAgent，无需修改现有代码
- Provider 可在运行时热切换

### 6.3 安全要求
- API Key 仅存储在本地 .env 文件，不上传
- .env 文件加入 .gitignore
- 不在日志中打印 API Key

### 6.4 平台支持
- iOS（主要）
- Android（主要）
- macOS（开发调试用）
- Web（受限，WebRTC 音频支持有限）

## 7. 术语表

| 术语 | 说明 |
|------|------|
| STT | Speech-to-Text，语音识别 |
| TTS | Text-to-Speech，语音合成 |
| LLM | Large Language Model，大语言模型 |
| STS | Speech-to-Speech，端到端语音对话 |
| Provider | 具体的 AI 服务实现（某个厂商的某个服务）|
| Agent | 编排多个 Provider 完成业务逻辑的服务对象 |
| chat-agent | 负责对话场景的 Agent |
| translate-agent | 负责翻译场景的 Agent |
